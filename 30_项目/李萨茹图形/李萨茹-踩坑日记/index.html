    踩坑日记 — 李萨茹图形测量仪调试实录

    坑1：相位差测量大角度跳变 — 50度到310度的幽灵

    现象

    李萨茹图形在相位差跨越 180度 时出现剧烈跳变——显示的相位突然从 179度 跳到 0度，或者从 5度 跳到 310度。图形也随之撕裂。

    排查过程

    1. 代码审查
    2. 原始代码直接用 (y_capture - x_capture) 计算时间差
    3. TIM1 的计数器是 16 位（最大 65535），到达后回绕到 0

    4. 当 Y 信号捕获值恰好跨越计数器回绕点（X在65500, Y在100），差值就会变成巨大的负数

    5. 问题复现 

      正常情况:  Y=30000, X=20000  -> diff = 10000 ticks (正确)
回绕情况:  Y=100,   X=65500  -> diff = 100 - 65500 = -65400 (错误!)

    6. 根因

    7. 简单的 (int32_t)(y - x) 无法处理 16 位计数器溢出回绕
    8. 大角度（>180度）时差值符号反转，显示相位 = 360 - 实际相位

    解决方案

    // 修正前（错误)
int32_t diff = (int32_t)(y_time - x_time);

// 修正后（16位自然溢出 + 取模)
int16_t diff = (int16_t)(y_time - x_time);  // 16位有符号自动处理溢出
int32_t delta_ticks = (int32_t)diff % (int32_t)p_ticks;
if (delta_ticks < 0) {
    delta_ticks += p_ticks;  // 负数取模修正
}

    原理：(int16_t)(y - x) 利用 C 语言有符号整数溢出特性：当 y=100, x=65500 时，65500 作为 int16_t 是 -36，100 - (-36) = 136，取模后得到正确的时差。

    经验教训

    定时器计数器会回绕！uint16_t 差值需转 int16_t 再处理，否则大角度相位会"倒立"。

    坑2：SPI 片选竞争 — MCP41010 双路互相干扰

    现象

    单独设置 U1 或 U2 正常，但双路同时输出时，其中一路的阻值随机漂移，相位不稳定。

    排查过程

    1. 示波器抓 SPI 总线
    2. SCK + MOSI 波形正常

    3. 但发现 CS1(PA4) 和 CS2(PA3) 有时同时为低

    4. 代码审查 

      void MCP41xx_1writedata(uchar dat1) {
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_3, GPIO_PIN_SET);  // 释放CS2
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_RESET); // 拉低CS1
    HAL_SPI_Transmit(&hspi1, spi_data, 2, 10);            // 传输
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_SET);   // 释放CS1
}

    5. 逻辑正确：先释放对方再拉低自己

    6. 但 HAL_GPIO_WritePin 每条指令约 200ns，如果中断在此间隙发生...

    7. 根因

    8. 如果在释放 CS2 之后、拉低 CS1 之前的瞬间发生中断（如 TIM2 触控扫描），中断处理可能耗时 > 1us
    9. 此时两根 CS 线都处于高电平，SPI 总线空闲
    10. 若中断中碰巧操作了 SPI，就会发送垃圾数据到总线上

    解决方案

    void MCP41xx_1writedata(uchar dat1) {
    __disable_irq();  // 关中断保护片选切换原子性

    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_3, GPIO_PIN_SET);
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_RESET);
    HAL_SPI_Transmit(&hspi1, spi_data, 2, 10);
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_SET);

    __enable_irq();   // 恢复中断
}

    经验教训

    多片选的 SPI 操作需要关中断保护！CS 切换间隙如果被中断打断，可能导致总线冲突。

    坑3：MCP41010 校准表"打表" — 手工逐点测量的血泪

    现象

    输入目标角度 45度，实际移相器输出只有 38度——非线性误差高达 7度。

    排查过程

    1. 理论计算翻车
    2. 最初尝试用公式 R = R_total * sin(phase/2) 计算阻值
    3. 忽略了 MCP41010 的端到端电阻公差 (+-20%) 和非线性误差 (+-1 LSB)

    4. 忽略了 PCB 走线 + 并联电阻的额外影响

    5. 实测数据揭示真相 

      理论:  0度->0阶, 45度->64阶, 90度->128阶, 180度->255阶
实测:  0度->0阶, 45度->203阶, 90度->224阶, 180度->251阶
      完全不是线性关系！

    6. 根因

    7. 移相网络（全通滤波器）的相位-阻值关系是反三角函数，非线性的
    8. MCP41010 自身的端阻公差（10k-ohm 名义值，实际 8~12k）
    9. PCB 上 2.7k-ohm 并联分流 → 等效阻值进一步偏移

    解决方案

    放弃理论公式，手工逐点打表：

    // 60个点的实测校准表
const int16_t Raw_Deg_Table[] = {
    5, 6, 7, 10, 15, 20, 30, 40, 41, 43, 44, 45, ...
    175, 180
};
const uint8_t Raw_U1_Reg_Table[] = {
    0, 11, 20, 70, 100, 146, 178, 195, 197, 198, 199, 200, ...
    250, 251
};
const uint8_t Raw_U2_Reg_Table[] = {
    0, 11, 20, 70, 100, 146, 178, 195, 197, 198, 199, 200, ...
    250, 250        // U2末尾和U1不同! 独立校准
};

    两点间线性插值： 

    // 查找目标角度落在哪两个测定点之间
for (i = 0; i < ARR_SIZE - 1; i++) {
    if (target_degree == Raw_Deg_Table[i]) {
        // 精确命中
        final_reg = Raw_Reg_Table[i];
        break;
    }
    if (target_degree > Raw_Deg_Table[i] && target_degree < Raw_Deg_Table[i+1]) {
        // 线性插值
        int32_t numerator = (target - x0) * (y1 - y0);
        final_reg = y0 + (numerator + denominator/2) / denominator;  // 四舍五入
        break;
    }
}

    经验教训

    模拟电路的"理论公式"在现实中基本不成立。手工逐点打表 + 线性插值虽然"笨"，但比任何公式都准。而且两路芯片要独立校准——没有两片 MCP41010 是完全一样的。

    坑4：TIM3 图形刷新撕裂 — 在中断里画图

    现象

    李萨茹图形偶尔出现水平撕裂线，上半帧和下半帧相位不同步。

    排查过程

    1. 示波器监测 VSYNC
    2. LCD 刷新率 60Hz (16.7ms/帧)
    3. TIM3 中断频率 ~2Hz (500ms)

    4. 理论上不可能撕裂...

    5. Logic Analyzer 抓 LTDC 时序

    6. 发现 lcd_draw_line() 在中断中绘制时间约 15~20ms
    7. LCD 在此期间刷新了至少一帧

    8. 如果正好刷到画了一半的线条，就出现撕裂

    9. 根因

    10. UI_DrawIndependentLissajous() 直接在中断上下文中调用
    11. 绘制 240 条线段 + 十字准星，耗时 15~20ms
    12. LCD 60Hz = 每 16.7ms 刷新一次，可能正好刷到绘制中途

    解决方案

    // 方案1: 中断中只设标志，主循环中绘制
volatile uint8_t flag_draw_lissajous = 0;

// TIM3 回调
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM3) {
        flag_draw_lissajous = 1;  // 只设标志
    }
}

// 主循环
while (1) {
    if (flag_draw_lissajous && is_running) {
        flag_draw_lissajous = 0;
        UI_DrawIndependentLissajous(/* ... */);
    }
}

    或者使用 DMA2D 双缓冲（如本工程中已配置的 DMA2D），在垂直消隐期（VSYNC 中断）完成缓冲区切换。

    经验教训

    中断里只做轻量操作。画图这种耗时 15ms+ 的操作放主循环，中断里只设标志位。

    坑5：X9313 上电状态未知 — 32阶从哪开始？

    现象

    X9313 上电后阻值位置不确定。每次重新上电，移相结果都不同。

    排查过程

    1. X9313 数据手册说明：上电后抽头位置恢复为 EEPROM 中存储的值
    2. 但如果在开发过程中频繁上下电，EEPROM 中可能存了"垃圾值"
    3. 代码中的 X9313_Current_Step 初始化为 0，与实际物理位置不符

    解决方案

    void X9313_Init(void) {
    X9313_CS_L();
    X9313_INC_H();
    X9313_UD_H();

    // 上电时强制归零：拉到最低端
    X9313_UD_L();  // 方向: 向下(减阻)
    for (int i = 0; i < 32; i++) {  // 发32个脉冲确保到底
        X9313_INC_L();
        delay_us(5);
        X9313_INC_H();
        delay_us(5);
    }

    X9313_CS_H();
    HAL_Delay(20);  // EEPROM 写入时间

    X9313_Current_Step = 0;  // 软件同步
}

    经验教训

    EEPROM 型数字电位器上电必须先归零！软件记录的 Current_Step 必须和硬件物理位置同步，否则后续所有调节都从错误起点开始。
