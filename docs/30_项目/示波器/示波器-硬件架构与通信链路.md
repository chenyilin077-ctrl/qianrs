# 示波器 — 硬件架构与通信链路

## 波形采集链路

### ADC + DMA + TIM3 触发

```
TIM3(200kHz TRGO) --> ADC1 IN0(PA0) --> DMA2_Stream0 循环 --> adc_buff[1024]
```

| 参数 | 值 |
|------|-----|
| ADC | ADC1, 12bit, 右对齐 |
| 通道 | CH0 (PA0) |
| 时钟 | PCLK2/4 = 90MHz/4 = 22.5MHz |
| 采样时间 | 15 cycles (约 0.67us) |
| 触发源 | TIM3 TRGO (上升沿) |
| 触发频率 | 200kHz (5us周期) |
| DMA | DMA2_Stream0, 循环模式, 半字 |
| 缓冲 | adc_buff[1024], 半字对齐 |

> **为什么用 TIM3 触发而非连续模式？**
> 连续模式采样周期不稳定（受中断抖动影响）。用定时器 TRGO 触发能保证**严格等间隔采样**，这对 FFT 频谱分析是必须的。

### 电压转换

```c
// ADC raw (0~4095) --> 电压 mV (0~3300)
uint16_t volt_mv = (raw * 3300) / 4095;  // 整数运算，避免浮点
```

## 频率测量链路

### ETR 级联架构

```
                    +-----------+
PA12(ext signal) -->| TIM1 ETR  |  16bit 计数器
                    | 180MHz    |  每个脉冲 +1
                    | /1 presc  |  溢出 → TRGO
                    +-----+-----+
                          | ITR0 (内部触发线0)
                    +-----v-----+
                    | TIM2 从模式|  32bit 计数器
                    | EXTERNAL1 |  每个TIM1溢出 +1
                    | /1 presc  |
                    +-----------+

48bit 总计数 = (uint32_t)TIM2_CNT << 16 | TIM1_CNT
```

### 定时器配置

| 参数 | TIM1 (ETR计数器) | TIM2 (溢出累加器) | TIM5 (周期读取) |
|------|-----------------|-------------------|-----------------|
| 模式 | ETRMODE2 | Slave EXTERNAL1 | 内部时钟 |
| 引脚 | PA12 | 无(内部ITR0) | 无 |
| 预分频 | 0 | 0 | 8999 |
| 周期 | 65535 | 4294967295 (32bit) | 9999 |
| 中断 | 无(硬件计数) | 无(硬件计数) | 2Hz |
| 触发 | 外部信号下降/上升沿 | ITR0(TIM1_TRGO) | - |

### 频率计算（差值法）

```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM5) {  // 每500ms
        static uint32_t last_total_pulses = 0;

        // 拼出48bit绝对脉冲计数
        uint32_t low_16  = __HAL_TIM_GET_COUNTER(&htim1);
        uint32_t high_16 = __HAL_TIM_GET_COUNTER(&htim2);
        uint32_t current  = ((uint32_t)high_16 << 16) | low_16;

        // 差值法：当前总数 - 上次总数 = 500ms内的脉冲数
        uint32_t delta = current - last_total_pulses;
        last_total_pulses = current;

        // 频率 = delta / 0.5s = delta Hz (因为周期是500ms)
        final_frequency = delta;
    }
}
```

> **为什么用差值法？**
> 不清零计数器，而是记录两次读数的差值。这样即使计数溢出回绕，无符号减法自动处理，永远正确。

## TouchGFX 显示链路

### 波形渲染（Screen2View::handleTickEvent）

```c
// 每3帧(~50ms)刷新一次
if (++frame_count >= 3) {
    frame_count = 0;
    dynamicGraph1.clear();

    // 读取DMA当前写入位置
    uint16_t curr_ptr = 1024 - __HAL_DMA_GET_COUNTER(hadc1.DMA_Handle);

    // 200点波形 + 3点滑动平均
    for (int i = 0; i < 200; i++) {
        avg = (adc_buff[idx0] + adc_buff[idx1] + adc_buff[idx2]) / 3;
        dynamicGraph1.addDataPoint(avg_mv);
    }
}
```

### 峰值保持电压测量

```c
static float v_max_held = 0.0f;

// 扫1024点找最大值
uint16_t local_max = 0;
for (int i = 0; i < 1024; i++) {
    if (volt_mv[i] > local_max) local_max = volt_mv[i];
}

// 峰值保持：只跟涨不跟跌（或慢速衰减）
if (local_max > v_max_held) {
    v_max_held = local_max;      // 发现更高峰值，瞬间跟上
} else {
    v_max_held -= 10.0f;         // 没发现新峰值，缓慢衰减 10mV/tick
    if (v_max_held < local_max) v_max_held = local_max; // 下限保护
}
```
